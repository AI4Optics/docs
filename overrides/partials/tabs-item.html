Material 9 has no dropdown navigation tabs: a top-level tab whose first child is itself a section (e.g. "Applications") normally collapses to a single deep link. We override that one branch to render the tab as a hover/focus pull-down listing its child sections (the individual applications), so new applications added under "Applications" in mkdocs.yml appear in the menu automatically. All other tabs (DeepLens, End2endImaging, DiffTMM — whose first child is a leaf "Overview" page) keep Material's default single-link behavior.
